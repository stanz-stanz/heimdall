## robots.txt Handling

**N/A.** DNS queries are protocol-level lookups to public DNS resolvers, not HTTP requests to the target's web server. robots.txt does not govern DNS resolution. Every device on the internet performs DNS lookups as part of normal operation.

## Reasoning

dnsx performs standard DNS record lookups — the same queries that every browser, email client, and network device makes when connecting to a domain. DNS records (A, AAAA, MX, TXT, CNAME, NS) are public by design — they must be publicly queryable for the internet to function.

This is explicitly allowed under SCANNING_RULES.md Level 0: "DNS records — A, AAAA, MX, TXT, CNAME, NS, SOA. Public by design." dnsx is a batch wrapper around standard DNS queries.

The function:
1. Writes domain list to temp file (includes primary domains + discovered subdomains)
2. Runs dnsx as subprocess to query public DNS resolvers
3. Parses JSON output — each line contains resolved records per domain
4. Returns dict of domain → {a, aaaa, cname, mx, ns, txt}
5. Gracefully skips if dnsx is not installed (returns empty dict)
6. Does NOT send any HTTP requests
7. Does NOT probe any paths
8. Does NOT perform any Layer 2 or Layer 3 activity

All activity is within Layer 1 (reading publicly served DNS data).
